\chapter{CONDICIONES TÉCNICAS}

En este apartado se exponen los requerimientos que tanto los recursos materiales como humanos deben cumplir para la correcta realización del presente proyecto.

\section{Recursos materiales}
El presente proyecto no requiere de recursos materiales hardware específicos, si bien es necesario un ordenador con un procesador multicore a fín de hacer los tiempos de compilación en desarrollo más cortos. En cuanto a los recursos materiales software, se recurre al software libre para cubrir dichas necesidades. Así se utiliza el siguiente software:

Sistema operativo Debian GNU/Linux Wheezy, con los siguiente paquetes instalados:
\begin{itemize}
	\item \textbf{build-essential} Un conjunto de aplicaciones que incluye todo lo básico para compilar un programa
	\item \textbf{git} El sistema de control de versiones que se va a utilizar
\end{itemize}

\subsection{Recursos humanos}

Las personas que se encargan del diseño y desarrollo del presente proyecto deben cumplir con los perfiles aquí indicados:
\begin{description}
\item[Director del proyecto] Ingeniero de Telecomunicaciones con experiencia en dirección de proyectos similares. Será el encargado de supervisar los diseños y de que se cumpla el plan de trabajo.
\item[Diseñador] Ingeniero de Técnico de Telecomunicaciones con conocimientos en el área de telemática. Será el encargado de realizar el diseño de los módulos del proyecto y los planes de pruebas de validación.
\item[Programador] Programador con amplia experiencia de programación en C y gran conocimiento del kernel Linux. Se encargará de desarrollar los prototipos necesarios para realizar la validación funcional del diseño de los módulos.
\item[Administrativo] se encarga de resolver las cuestiones administrativas y burocráticas así como de la documentación del proyecto.
\end{description}

\subsection{Documentación}
A continuación se describen los documentos que deben utilizarse para la realización del proyecto. A pesar de ser un recurso material, se describe en un apartado diferente dada su importancia.

\begin{description}
\item [Documento de Memoria]
En este documento se exponen las razones que impulsan la realización de este proyecto. Así, en él se recogen los objetivos que persigue este proyecto así como los diferentes beneficios que reporta de ejecución. Se enumeran las diferentes alternativas que se dan para la realización del mismo y se escogen las opciones que se consideran adecuadas para conseguir un diseño óptimo y un cumplimiento eficiente de los objetivos. 

También se incluye un breve resumen y las especificaciones funcionales de los módulos del sistema. Para finalizar se adjunta un resumen del plan de trabajo y del presupuesto del presente proyecto.

\item [Documento de Diseño de Alto nivel]
Este documento describen las funcionalidades y especificaciones de alto nivel de las diferentes partes del proyecto, sirve como guía general para saber cual es el razonamiento de diseño que se ha seguido.

\item [Documento de Diseño de Bajo nivel]
Este documento ofrece, desde un punto de vista técnico, una descripción detallada del modo en que se ha de implementar el proyecto. Se explica la forma que toma la solución general explicada en el documento de Alto nivel, así como ejemplos de código que sirvan al Programador para guiarse.

\item [Documento de Pliego de condiciones]
El presente documento.
\end{description}
